# 小酥存货系统 - 前后端分离开发

## 目标
将纯静态存货展示站升级为前后端分离系统，支持玩家下单 + 管理员订单管理

## 完成内容

### 后端 (server.js)
- Express 服务器，端口 18890
- JSON 文件存储 (db.json)，自动从 data.js 加载 86 服库存
- RESTful API：库存查询/更新、订单 CRUD
- 管理密码保护：`xiasu666`
- 库存校验：下单时自动检查库存是否充足
- 模糊服务器名匹配

### 玩家端 (public/index.html)
- 赛博霓虹风格 UI，支持搜索 + 大区筛选
- 86 服库存展示（可折叠卡片）
- 一键下单弹窗：选区服 → 填玩家名 → 填数量 → 确认
- 自动库存校验，超出库存提示

### 管理端 (public/admin.html)
- 密码登录保护
- 订单管理：按状态筛选（待发货/已发货/已取消）、搜索
- 操作：发货、取消、撤回、删除
- 一键导出 Excel（格式：区服, 收货玩家, 物品ID, 数量, 平台版本）
- 30 秒自动刷新

### 测试验证
- ✅ 服务启动成功，86 服数据加载
- ✅ API 下单测试通过
- ✅ API 删除订单测试通过
- ✅ 页面 HTML 正常返回

## 访问
- 玩家端：http://localhost:18890
- 管理端：http://localhost:18890/admin.html
- 启动方式：双击 `inventory/启动.bat`
